# 1단계. 더웰파트너 관리자(Admin) 전체 시스템 설계

## 1. 관리자 시스템 목적

관리자 페이지는 단순 게시판 관리가 아니라 아래 업무를 한 곳에서 처리할 수 있어야 합니다.

* 홈페이지 콘텐츠 관리
* 채용공고 등록 및 지원자 관리
* 기업 문의 접수 및 상담 관리
* 임직원 서류발급 관리
* 직원/퇴직자 정보 관리
* 향후 근태·급여·배치 시스템 확장 기반 마련

즉, 목표는 **홈페이지 관리자 + 채용 관리자 + 인사서류 관리자**를 합친 구조입니다.

## 3. 관리자 권한 구조

관리자는 역할별로 나누는 것이 좋습니다.

### 3-1. 최고관리자

전체 메뉴 접근 가능
설정, 권한, 직원정보, 서류발급, 채용, 홈페이지 관리 전부 가능

### 3-2. 홈페이지 관리자

* 메인/서브페이지 수정
* 배너/공지/FAQ 관리
* 고객사 로고 관리

### 3-3. 채용 관리자

* 채용공고 등록
* 지원자 조회
* 상담 상태 변경
* 인재풀 관리

### 3-4. 인사/서류 관리자

* 직원정보 관리
* 퇴직자 관리
* 재직증명서/퇴직증명서 템플릿 관리
* 급여명세서/원천징수 업로드
* 발급 이력 조회

### 3-5. 조회 전용 관리자

* 데이터 열람만 가능
* 수정/삭제 불가

# 2단계. 아웃소싱 ERP 구조 설계

## 1. ERP 목적

홈페이지가 외부 홍보 채널이라면, ERP는 내부 운영의 핵심입니다.

목표:

* 근로자 배치 관리
* 현장별 인원 운영
* 출근/결근/교대 관리
* 급여 계산 기초데이터 관리
* 현장관리자와 본사 협업 강화

---

## 2. ERP 핵심 메뉴 구조

```text
ERP
├ 인력관리
├ 현장관리
├ 배치관리
├ 출퇴근/근태관리
├ 급여기초관리
├ 문서관리
├ 통계/리포트
└ 설정
```
