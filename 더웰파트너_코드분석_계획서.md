## 1. 프로젝트 개요

### 1.1 프로젝트 성격
- **공개 사이트**: 생산 아웃소싱 / HR 전문 기업 홈페이지 (채용, 서비스 안내, 문의)
- **어드민 시스템**: 채용관리, ERP, 급여, 외국인근로자, 문서발급, 통계 등 종합 관리 시스템
- **이중 구조**: 공개 페이지(12개) + 관리자 페이지(19개)

### 1.2 기술 스택
| 구분 | 기술 | 버전 |
|------|------|------|
| 프레임워크 | React | 18.3.1 |
| 빌드 도구 | Vite | 6.3.5 |
| 라우팅 | React Router | 7.13.0 |
| UI 라이브러리 | shadcn/ui (Radix UI) | 40+ 컴포넌트 |
| 스타일링 | Tailwind CSS | 4.1.12 |
| 애니메이션 | Motion (Framer Motion) | 12.23.24 |
| 차트 | Recharts | 2.15.2 |
| 폼 | React Hook Form | 7.55.0 |
| 백엔드 | Supabase Functions | - |
| 아이콘 | Lucide React | 0.487.0 |
| 추가 | Material UI, jsPDF, date-fns, react-dnd | - |

### 1.3 아키텍처 구조
```

├── app/
│   ├── App.tsx                       # 라우터 래퍼
```

## 3. 현재 상태 평가

### 3.1 잘 설계된 부분 (강점)

| # | 항목 | 평가 |
|---|------|------|
| 1 | **중앙화된 API 클라이언트** | api.ts에서 모든 API 호출 관리 - 일관성 높음 |
| 2 | **Lazy Loading** | 모든 페이지가 `lazy(() => import())` 적용 |
| 3 | **컴포넌트 분리** | 공개/관리자 레이아웃 명확히 분리 |
| 4 | **UI 컴포넌트** | shadcn/ui 40+ 재사용 가능 컴포넌트 |
| 5 | **SEO Hook** | useSEO로 동적 메타태그, OG 태그 관리 |
| 6 | **회사정보 Context** | CompanyInfoContext로 전역 상태 관리 |
| 7 | **CSV Export** | 스키마 기반 CSV 내보내기 유틸리티 |
| 8 | **감사 로깅** | 관리자 활동 추적 시스템 |
| 9 | **카카오톡 연동** | 급여명세서 카카오 발송 기능 |
| 10 | **배지 카운트** | 사이드바 실시간 미처리 건수 표시 |

### 3.2 발견된 문제점

#### [치명적] Critical Issues

| # | 문제 | 영향도 | 위치 |
|---|------|--------|------|
| C1 | **XSS 취약점**: localStorage에 토큰 저장 | 높음 | api.ts, AdminLayout |
| C2 | **CSRF 보호 없음** | 높음 | 전체 API 호출 |
| C3 | **로그인 Rate Limiting 없음** | 높음 | AdminLogin |
| C4 | **Public Anon Key 하드코딩** | 중간 | utils/supabase/info.tsx |
| C5 | **IP 추적 미구현**: 감사 로그 IP가 항상 "browser" | 중간 | api.ts |

#### [중요] Major Issues

| # | 문제 | 영향도 | 위치 |
|---|------|--------|------|
| M1 | **데이터 캐싱/페칭 라이브러리 없음**: 모든 페이지가 raw useEffect + fetch | 높음 | 전체 페이지 |
| M2 | **폼 유효성 검증 불일치**: 수동 검증, 스키마 없음 | 중간 | Contact, Recruit, Register |
| M3 | **대형 컴포넌트**: AdminPayroll 1500+줄, AdminForeign 1200+줄 | 중간 | admin/ 폴더 |
| M4 | **에러 바운더리 부족**: Root 레벨에만 1개 | 중간 | pages/ |
| M5 | **상태 관리 한계**: 복잡한 useState 체인, useReducer 미사용 | 중간 | admin/ 페이지들 |

#### [개선] Improvement Items

| # | 문제 | 영향도 | 위치 |
|---|------|--------|------|
| I1 | **매직 넘버/문자열**: 상태 라벨 중복 하드코딩 | 낮음 | 여러 페이지 |
| I2 | **일관성 없는 에러 처리**: toast/alert/console.error 혼용 | 낮음 | 전체 |
| I3 | **하드코딩 색상값**: 일부 inline style에 직접 색상 | 낮음 | 컴포넌트들 |
| I4 | **반응형 테이블 미최적화** | 낮음 | admin/ 테이블들 |
| I5 | **이미지 최적화 없음** | 낮음 | 공개 페이지 |
| I6 | **공유 Hook 폴더 없음**: hooks가 lib에 혼재 | 낮음 | lib/ |
| I7 | **상수/열거형 파일 없음** | 낮음 | 전체 |

## 7. 기술 부채 현황 요약

| 카테고리 | 현재 상태 | 목표 상태 | 위험도 |
|----------|-----------|-----------|--------|
| 보안 | localStorage 토큰, CSRF 없음 | HttpOnly Cookie, CSRF 토큰 | 높음 |
| 데이터 페칭 | raw useEffect + fetch | TanStack Query | 중간 |
| 폼 검증 | 수동 / 불일치 | Zod + react-hook-form | 중간 |
| 에러 처리 | toast/alert 혼용 | 통합 에러 핸들러 | 중간 |
